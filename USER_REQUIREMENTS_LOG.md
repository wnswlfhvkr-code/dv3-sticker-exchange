## 📅 [2026-06-05] 피드백 및 요구사항 (진행 예정)

1. **🖼️ 카테고리 사진 마우스 호버 시 포토샵 툴팁 수정**
   - *의도/현상*: 카테고리 사진에 마우스를 올리면 브라우저 단에서 "Adobe Photoshop" 이라는 원치 않는 툴팁이 뜸.
   - *해결 방안*: `<img>` 태그에 명시적인 `title` 속성을 매핑하여 이미지 파일의 메타데이터가 임의 노출되는 현상을 해결함.

2. **🔑 로그인 유저별 데이터 소유권 분기 및 삭제 권한 오류 수정**
   - *의도/현상*: `간장` 계정으로 로그인했는데 내 글이 아닌 타인(`sd`)의 글이 '내 글'로 표시되고 삭제 권한이 잘못 노출되는 현상.
   - *원인*: 브라우저 로컬스토리지를 공용 배열로 체크하여 권한 꼬임 발생.
   - *해결 방안*: 로그인한 계정의 `userNickname`과 각 게시글의 `post.nickname`이 정확히 일치할 때만 삭제 권한을 부여하도록 획기적으로 로직을 개편함.

3. **🚫 3x3 카드 도감의 대표 이미지 클릭 방지**
   - *의도/현상*: 카테고리 진입 후 우상단의 대표 이미지 동그라미를 누르면 원본 사진 파일 주소로 넘어가서 사용성을 해침.
   - *해결 방안*: 대표 이미지 클릭 시 이동하는 이벤트(`window.open` 등)를 완전히 제거하고 순수 장식용 이미지로 변경함.

4. **📞 연락처 정보 선택 기입화 및 연락하기 조건부 렌더링**
   - *의도/현상*: 글 등록 시 연락처를 필수로 적어야만 글이 써지는 제한이 불편함. 기입하지 않고 등록했을 때는 피드에 '연락하기' 버튼이 노출되지 않도록 함.
   - *해결 방안*: 등록 폼의 필수 입력을 제외하고, 피드 카드 렌더링 시 연락처 텍스트가 실제로 존재할 때만 '연락하기' 링크 버튼이 조건부 노출되도록 개선함.

5. **📂 사이드 "내 정보" 탭 탑재 (Drawer UI)**
   - *의도/현상*: 내가 올린 교환 글 목록만 빠르게 한눈에 확인하고, 간편하게 내 연락처 정보를 업데이트하거나 글을 삭제할 수 있는 슬라이딩 사이드 서랍 메뉴를 요구함.
   - *해결 방안*: 헤더 닉네임 옆에 정보 확인 버튼을 제공하고, 오른쪽에서 연두색 네온 스타일의 슬라이딩 드로어가 열리도록 구현함.
