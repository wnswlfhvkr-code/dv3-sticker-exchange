## 📅 [2026-06-06] 피드백 및 요구사항 (v4.4 완료)

1. **📋 관리자용 조치 완료 신고 로그(Log) 시스템 추가 [LATEST]**
   - *의도/현상*: 관리자가 신고 목록을 확인하고 반려하거나 글/댓글을 삭제하여 사건을 종결했을 때, 종결된 내용이 즉시 사라져 어떤 조치를 취했었는지 사후에 모니터링하거나 로그로 확인할 수 없음.
   - *해결 방안*: 신고 조치 시 종결된 내역을 로컬스토리지 `dv3_admin_resolved_logs` 에 조치 일시, 조치 유형(반려/삭제), 대상 유형, 신고자, 사유, 상세 내용을 누적 기록하는 시스템을 설계했습니다. 관리자 전용 대시보드 내에 **🚨 신고 접수**와 **📋 조치 완료 로그** 분할 탭을 제공하여, 사후 조치 내역을 손쉽게 모니터링할 수 있도록 구현했으며, 전체 로그를 비울 수 있는 기능도 탑재 완료했습니다.

2. **💬 실시간 1:1 채팅방 목록 동기화 및 선택방 액티브 하이라이트 매칭 완료 [LATEST]**
   - *의도/현상*: 피드에서 특정 유저 글의 1:1 채팅을 눌러 대화창에 정상적으로 진입했으나, 좌측 채팅방 목록에서 현재 대화 중인 방이 어떤 방인지 시각적으로 구분이 불가능함. 또한, 상대방이 실시간으로 메시지를 보낼 때도 목록의 마지막 메시지와 시각 및 정렬 순서가 즉시 업데이트되지 않는 현상이 발생함.
   - *해결 방안*:
     - **목록 네온 하이라이팅**: 채팅 목록을 그릴 때 현재 입장한 활성 대화방(`room.id === chatActiveRoomId`)인 경우 연두색 네온 테두리(`border: 1px solid var(--primary-color)`)와 은은한 연두색 투명 네온 광채 배경을 부여하여 비주얼 일체감을 확보했습니다.
     - **실시간 목록 갱신**: 글로벌 새 메시지 구독 리스너(`subscribeAllMyMessages`) 최상단에 `loadChatRooms()`를 삽입하여, 상대방이 메시지를 보낼 때마다 목록이 즉시 새로고침되도록 조치했습니다.
     - **탭 간 실시간 목록 동기화**: 로컬스토리지 갱신 이벤트(`dv3_chat_update`) 리스너를 `App.jsx`에 이펙트로 추가하여, 로컬 저장소 상에 메시지 변동이 감지되는 즉시 대화방 목록도 탭 단위로 초고속 갱신 동기화되도록 개선했습니다.

## 📅 [2026-06-06] 피드백 및 요구사항 (v4.3 완료)

1. **🛡️ 신고 및 댓글 기능 Supabase DB 테이블 누락 대비 자동 로컬 폴백(Fallback) 구현**
   - *의도/현상*: Supabase 데이터베이스 상에 `reports` 및 `post_comments` 테이블이 없어서 신고 버튼 클릭 시 `"Could not find the table 'public.reports' in the schema cache"` 에러 팝업창이 뜨며 기능이 중단됨.
   - *해결 방안*: 관련 6개 DB 통신 API(`fetchComments`, `addComment`, `removeComment`, `addReport`, `fetchReports`, `resolveReport`)에 try-catch 예외 래퍼 및 테이블 부재 오류 판별식을 장착했습니다. Supabase에서 테이블이 발견되지 않으면 즉각 LocalStorage 시뮬레이터(`mockDB`)로 실행 경로를 우회시킴으로써, 사용자가 신고나 댓글을 작성해도 에러 없이 100% 정상적으로 작동하고 관리자가 제재할 수 있도록 보안 처리했습니다. 향후 테이블이 추가되면 자동으로 원격 연동 모드로 자동 스위칭됩니다.

2. **💬 수신/발신 1:1 통합 유일 채팅방 구현 및 다중 생성 차단**
   - *의도/현상*: 한 사람과 대화할 때 올린 글마다 대화방 창이 마구 쪼개져 여러 개 열리는 버그가 발생하여, 나와 상대방 간에는 글 ID와 무관하게 유일한 단 하나의 1:1 통합 대화방만 연결되도록 요구함.
   - *해결 방안*: `buyerNickname` 과 `sellerNickname` 을 가나다순으로 자동 정렬하여 고유한 식별 방 ID(`room-닉네임A-닉네임B`)를 부여하도록 `chatService.js` 내부 로직을 전면 수정했습니다. 이로 인해 누가 먼저 말을 걸었는지(수발신 역할) 및 피드 글 ID(`postId`)와 상관없이 한 대화 상대방과는 무조건 단 하나의 채팅방으로 연결되도록 완벽히 통합했습니다.

3. **🔔 실시간 글로벌 메시지 알림 및 미리보기 팝업 완벽 복구**
   - *의도/현상*: 대화가 도착해도 띵동 알림음이 울리지 않고, 우하단 토스트 미리보기 알림 팝업 및 안 읽은 네온 배지가 활성화되지 않는 현상이 발생함.
   - *해결 방안*: Supabase 모드에서 1:1 통합 방 ID(`roomId`)가 닉네임 조합 텍스트 형태로 테이블에 직접 쓰이도록 insert 구문을 보강하여, 실시간 알림 리스너의 `room_id.includes(myNickname)` 조건 검증이 100% 정상 작동하도록 조치했습니다. 알림음을 재생하고 우하단 실시간 토스트 및 안 읽은 개수를 실시간으로 그려줍니다. 토스트 클릭 시 해당 통합 닉네임 방으로 맞물리도록 닉네임 디코딩 파서도 동치 수정했습니다.

4. **🧹 구버전 대화방 파편화 로컬 데이터 자동 정화 마이그레이션**
   - *의도/현상*: 1:1 통합 대화방 체계로 개편되면서 로컬스토리지에 기존 규격(`room-글ID-구매자-판매자`)의 파편화된 구버전 대화 기록이 남아있을 경우 렌더링 충돌이 유발됨.
   - *해결 방안*: `App.jsx` 최초 실행 시 `localStorage` 에 남아있는 구버전 4세그먼트 대화방 캐시 데이터를 자동 필터링하여 영구히 클린 제거해 주는 마이그레이션 함수를 이펙트에 삽입했습니다.

5. **🏷️ 게스트 계정 닉네임 " (게스트)" 표기 정책 적용**
   - *의도/현상*: 게스트로 로그인한 사용자가 임시 계정임을 명확하게 인지하고 타인들이 식별할 수 있도록 게스트 표시를 요구함.
   - *해결 방안*: 게스트 로그인 시 세션 및 유저 상태 닉네임에 ` (게스트)` 문자열을 자동 덧붙이도록 처리하여, 피드 목록의 글 카드, 댓글 목록, 1:1 채팅방 및 대화방 목록, 실시간 온라인 접속자 현황(`onlineUsers`) 전체에 걸쳐 항상 `닉네임 (게스트)` 로 자동 출력되도록 개선했습니다. 정식으로 회원가입 후 암호화 로그인한 회원들과 명확하게 투명하게 분리됩니다.

6. **🔐 게스트 로그인 계정 도용 차단 및 중복 닉네임 검증 구현**
   - *의도/현상*: 비밀번호 없이 임시로 들어오는 게스트가 가입 회원의 닉네임(예: `간장` 등)으로 로그인할 경우, 기존 계정의 글 소유권이나 대화 권한이 임시 게스트에게 노출되거나 도용될 위험이 발생함. 또한 닉네임이 똑같은 두 게스트가 접속했을 때 닉네임 겹침 오작동이 일어남.
   - *해결 방안*: 게스트 로그인 시도 시 `dbService.verifyUser` 조회를 실행하여 정식 회원에 이미 등록된 닉네임인지 확인하고, 회원 닉네임일 경우 게스트 로그인을 강력 차단하고 정식 회원 비밀번호 로그인을 유도하도록 경고를 띄웁니다. 추가로 현재 실시간 온라인 리스트(`onlineUsers`)를 체크하여 닉네임이 겹치는 다른 접속자가 있는 경우 로그인을 불허하도록 처리 완료했습니다.

7. **🛠️ 내 글 수정 기능 (Edit Post)**
   - *의도/현상*: 내가 올린 교환 글을 지우고 다시 쓰는 번거로움을 해결하기 위해 본인 글 수정 기능을 요구함.
   - *해결 방안*: 피드 카드 및 내 정보 사이드 드로어 내에 '수정' 버튼을 추가하고, 수정 모달창을 개발했습니다. 수정 모달 내에 연락처 변경 기입란 뿐만 아니라 줄 수 있는 카드와 받고 싶은 카드를 모달 안에서 편하게 터치하여 실시간으로 가감(추가/제거)할 수 있는 직관적인 컴팩트 스티커 도감 바구니 셀렉터를 함께 설계하여 사용자 경험을 극대화했습니다.

8. **💬 댓글 기능 (Post Comments)**
   - *의도/현상*: 교환 글 피드 카드 내에서 다른 유저들과 자유롭게 피드백을 주고받거나 예약 상태 등을 논의할 수 있는 댓글 영역을 요구함.
   - *해결 방안*: 카드 하단에 '💬 댓글 (N)' 아코디언 버튼을 제공하여 토글이 가능하도록 만들었습니다. 댓글 목록 조회, 실시간 댓글 등록 및 댓글 작성자 본인/관리자 전용 삭제 권한 핸들러를 완벽 매핑했습니다.

9. **🚨 신고 기능 (Report / Flag)**
   - *의도/현상*: 허위 매칭 글, 사기 의심 연락처, 욕설 및 비방 댓글 등을 운영진에게 신고하여 정화할 수 있는 수단을 요구함.
   - *해결 방안*: 피드 카드 내에 빨간색 사이렌 신고 아이콘과 댓글 옆에 신고 버튼을 배치하고, 신고 사유(허위 정보, 사기 의심, 욕설/비방, 기타 수기 작성 등)를 선택 및 입력할 수 있는 신고 모달창을 추가하여 관리 시스템으로 인계되도록 구현했습니다.

10. **👑 관리자 전용 대시보드 구축 및 `간장` 계정 관리자 지정**
    - *의도/현상*: 닉네임 `간장` 로그인 시 사이트를 관리할 수 있는 전용 기능 및 신고 내역 모니터링 관리 도구를 요구함.
    - *해결 방안*: 로그인한 닉네임이 `간장`일 경우 헤더 닉네임 우측에 👑 관리자 배지가 추가로 표시되며, 클릭 시 전용 대시보드 모달이 오픈됩니다. 대시보드에서는 유저들이 신고한 실시간 누적 신고 목록을 모니터링하고, 원클릭으로 '신고 반려(무시)' 처리하거나 문제가 되는 게시글/댓글을 '강제 삭제 & 종결' 제재 조치할 수 있도록 강력한 중앙 관리 시스템을 완벽 구축했습니다.
