## 📅 [2026-06-06] 피드백 및 요구사항 (v4.8 완료)

1. **🔒 황금테두리(isGolden) 스티커 교환 격리 차단**
   - *의도/현상*: 황금테두리 스티커는 게임 내에서 "교환 불가"이므로 거래하거나 바구니에 담을 수 없도록 완전 비활성화해야 함.
   - *해결 방안*: 
     - 도감 화면 상에서 황금 카드는 선택 불가능하게 `cursor: not-allowed` 처리하고, `opacity: 0.3`, `grayscale` 필터를 입혀 시각적 대비를 주었습니다.
     - 카드 위에는 회색 자물쇠(`🔒`) 배지와 하단 붉은색 그라디언트 `"교환 불가"` 배너를 오버레이 렌더링했습니다.
     - `toggleStickerSelection` 및 `toggleEditStickerSelection` 첫 단에 차단 가드를 이식해 선택을 불허하고, 원격/로컬 DB 연동 시에도 `isGolden` 항목은 원천 배제(filter)되도록 조치했습니다.

2. **⚡ 도감(메인) 선택 시 내 등록 글 실시간 자동 수정(업데이트) 연동**
   - *의도/현상*: 이미 작성한 교환글이 존재할 때, 메인 도감 화면에서 변경한 사항이 바로 연동되지 않고 수정 모달로 다시 들어가 수동으로 진행해야 하는 불편함 해소.
   - *해결 방안*: 내 글이 존재하는 사용자가 메인 도감 슬롯을 누를 때마다, 바구니 상태 갱신 직후 `updatePostStickersDirectly` 헬퍼 함수를 기동하여 Supabase 원격 DB와 로컬 피드 posts 배열의 관련 글 스티커 목록이 지연 없이 즉각 동적 업데이트되도록 조치했습니다.

3. **📦 피드 카드 내 교환 리스트 접이식 아코디언 컴포넌트 이식 및 기본 닫힘 적용**
   - *의도/현상*: 여러 카드를 올려둔 글이 많은 경우 스크롤이 지나치게 길어져 시각적 피로 및 탐색 불편 초래.
   - *해결 방안*: 피드의 모든 포스트 내 스티커 리스트를 세련된 접이식 UI로 구현하고 기본 닫힘(`expandedPostIds: []`) 상태로 제공합니다. `📦 교환 카드 목록 (총 개수) 자세히 보기 ▼` 버튼 클릭 시에만 리스트가 안전하게 확장됩니다.

4. **🛠️ 글 수정 모달 3x3 도감식 격자 뷰 개편 및 PC 단축키 바인딩**
   - *의도/현상*: 기존의 5열 가로 리스트형 수정 방식이 메인 화면(도감 3x3)에 비해 매우 낯설고 불편함.
   - *해결 방안*: 
     - 수정 모달의 하단부 스티커 선택 셀렉터를 메인 도감 뷰 레이아웃과 100% 동일하게 ◀ ▶ 이동 단추 및 3x3 격자도감 형태로 이식했습니다.
     - 메인과 마찬가지로 마우스 좌클릭 시 보유 등록, 우클릭 시 필요 등록이 한 창에서 가능하도록 좌우클릭 이벤트를 통합 탑재했습니다.
     - 키보드 `4`번(이전 카테고리) 및 `6`번(다음 카테고리) 단축키 리스너를 전역 바인딩하여, 사용자가 input에 글을 쓰는 경우가 아니라면 마우스 없이도 페이지 전환이 초고속으로 가능하게 설계하고 화살표 옆에 `(단축키 안내)`를 명기했습니다.

## 📅 [2026-06-06] 피드백 및 요구사항 (v4.6 완료)

1. **🔄 내 교환글 스티커 카드 수정 시 상단 바구니 상태 실시간 동기화 연동 완료**
   - *의도/현상*: 사용자가 내 교환 게시글의 스티커 정보를 '수정' 했음에도 불구하고, 화면 상단의 내 바구니 상태(`myHaves`, `myWants`)가 기존 정보 그대로 유지되어, 피드에서 다른 사람들의 카드 정보와 매칭을 판단할 때 옛날 기준을 바탕으로 매칭 배지가 계산되던 오동작이 유발됨.
   - *해결 방안*:
     - **실시간 수정 동기화**: `handleUpdatePost` 에서 글 수정을 완료하는 즉시 변경된 `editHaves` 와 `editWants` 데이터를 내 상단 바구니 상태(`myHaves`, `myWants`) 및 로컬스토리지 백업 데이터에 실시간으로 업데이트하게 수정했습니다.
     - **글 리스트 로딩 시 바구니 동기화 헬퍼 (`syncMyBasketFromPost`) 이식**: 1.5초 주기 폴링 리로드(`fetchData`, `fetchDataSilent`) 시점에, 로그인한 유저가 등록해 둔 게시글 스티커 정보가 발견되면 그 글의 스티커 정보를 기반으로 화면 상단 바구니 데이터를 실시간 싱크(동기화)되도록 처리하여, 어떤 탭이나 기기에서 수정하든 무조건 변경된 카드를 기준으로 매칭 연산이 즉각 재작동하도록 완성했습니다.

   - *의도/현상*: 데이터베이스(Supabase) 및 로컬스토리지에 예전에 생성되었던 구버전 방 레코드가 지워지지 않고 잔존하고 있어, 채팅 목록 조회 시 동일 유저와의 대화방이 2개 이상 중복 노출됨.
   - *해결 방안*:
     - **원격/로컬 DB 통합 마이그레이터 이식**: 사용자가 로그인할 때 활성화되는 `runDatabaseMigration(userNickname)` 모듈을 추가하여, 구버전 방 규격을 자동 추출하고 모든 대화 메시지들(`chat_messages` 테이블 내 `room_id`)을 최신 1:1 통합 방 ID(`room-닉네임A-닉네임B`)로 안전하게 업데이트(이관) 처리합니다.
     - **구버전 방 레코드 제거**: 메시지가 전부 최신 통합 방으로 옮겨지면, 기존의 구버전 방 레코드는 DB 및 로컬스토리지에서 영구히 삭제(Delete) 처리하여 한 대화 당사자 간에는 **결국 단 하나의 대화방만 존재**하게 정화했습니다.

2. **💬 실시간 1:1 채팅방 목록 동기화 및 선택방 액티브 하이라이트 매칭 완료**
   - *의도/현상*: 피드에서 특정 유저 글의 1:1 채팅을 눌러 대화창에 정상적으로 진입했으나, 좌측 채팅방 목록에서 현재 대화 중인 방이 어떤 방인지 시각적으로 구분이 불가능함.
