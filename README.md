# 🐉 드래곤 빌리지 3 스티커(카드) 교환소

드래곤 빌리지 3 유저들이 서로의 중복 스티커를 확인하고, **자동으로 100% 매칭되는 최적의 교환 파트너**를 찾아 연락할 수 있는 웹 플랫폼입니다.

이 프로젝트는 초보자분들도 쉽게 실행하고 Vercel에 배포할 수 있도록 구성되어 있습니다.

---

## ✨ 주요 기능

1. **내 교환 프로필 설정**: 내 닉네임, 연락처(오픈챗 등), 내가 가진 카드와 구하는 카드를 입력할 수 있습니다.
2. **실시간 교환 게시판**: 다른 유저들이 올린 교환 글을 한눈에 볼 수 있습니다.
3. **⚡ 100% 자동 매칭**: 내 프로필을 입력하면, 게시판의 유저들 중 나와 서로 딱 맞게 스티커를 교환할 수 있는 사람이 누군지 분석하여 하이라이트 해줍니다.
4. **하이브리드 데이터베이스**: 별도 설정 없이 켜면 브라우저 저장소(LocalStorage) 모드로 동작하며, `.env` 설정 시 전세계 유저와 공유할 수 있는 실시간 Supabase 데이터베이스 모드로 전환됩니다.

---

## 🚀 로컬에서 실행하는 방법

아주 간단하게 컴퓨터에서 실행해볼 수 있습니다.

1. **패키지 설치** (최초 1회 실행)
   ```bash
   npm install
   ```

2. **개발 서버 실행**
   ```bash
   npm run dev
   ```
   * 터미널에 나오는 `http://localhost:5173` 주소를 브라우저로 열면 됩니다!

## ☁️ Supabase 온라인 데이터베이스 연동하기 (배포 필수)

전세계 사람들과 글을 공유하려면 온라인 데이터베이스를 연결해야 합니다.

1. **[Supabase](https://supabase.com/)**에 접속하여 무료 계정으로 가입하고 새 프로젝트를 생성합니다.
2. 프로젝트 대시보드의 **Settings -> API**에서 `Project URL`과 `Anon Key`를 복사합니다.
3. 프로젝트 폴더 루트(가장 바깥)에 `.env` 파일을 생성하고 아래와 같이 붙여넣습니다:
   ```env
   VITE_SUPABASE_URL=여기에_Project_URL_붙여넣기
   VITE_SUPABASE_ANON_KEY=여기에_Anon_Key_붙여넣기
   ```
4. Supabase 대시보드의 **SQL Editor**로 이동하여 **New Query**를 누른 뒤 아래 SQL을 붙여넣고 **Run**을 실행해 테이블을 만듭니다:
   ```sql
   create table posts (
     id bigint generated by default as identity primary key,
     nickname text not null,
     contact text not null,
     haves text[] default '{}',
     wants text[] default '{}',
     created_at timestamp with time zone default timezone('utc'::text, now()) not null
   );

   -- 누구나 읽고 쓸 수 있도록 RLS 비활성화 (개발 편의성용)
   alter table posts disable row level security;
   ```

## 🌐 Vercel에 1분 만에 무료 배포하기

1. 이 프로젝트를 본인의 **GitHub 저장소(Repository)**에 올립니다.
2. **[Vercel](https://vercel.com/)**에 로그인하고 **Add New -> Project**를 누릅니다.
3. GitHub 저장소를 연결합니다.
4. **Environment Variables** 설정에 다음 두 개를 입력해 줍니다:
   - `VITE_SUPABASE_URL` = (본인의 Supabase URL)
   - `VITE_SUPABASE_ANON_KEY` = (본인의 Supabase Anon Key)
5. **Deploy** 버튼을 누르면 배포 끝! 생성되는 주소로 사람들과 공유하여 사용하세요.
