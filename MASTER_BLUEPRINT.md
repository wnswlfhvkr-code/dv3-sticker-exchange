## 📂 파일 구조 설명

이 프로젝트의 핵심 파일들은 다음과 같이 배치되어 있습니다:

- **`index.html`**
  - 사이트의 뼈대가 되는 파일입니다. 검색엔진 노출을 위한 정보(SEO)와 한국어 설정이 적용되어 있습니다.
- **`src/main.jsx`**
  - React 앱의 진입점으로, `App.jsx`와 `index.css`를 연결해 브라우저에 화면을 띄워주는 역할을 합니다.
- **`src/App.jsx`**
  - **애플리케이션의 핵심**입니다. 닉네임+비밀번호 기반 로그인/회원가입 제어, 복합 선택 바구니(Haves/Wants), 3x3 스티커 도감, 실시간 양방향 매칭 엔진, 그리고 1:1 채팅창 UI 및 액션 로직이 모두 집약되어 있습니다. 상세 도감 화면 상단에 바로 옆 카테고리로 1초 만에 스위칭할 수 있는 꺾쇠 내비게이션(`<`, `>`)이 내장되어 있습니다. 또한 온라인 접속 유저 목록(`onlineUsers`)을 실시간으로 렌더링해 줍니다. v4.3에서는 구버전 파편화 대화방 캐시 데이터를 최초 마운트 때 자동 감지해 싹 밀어주는 캐시 초기화 프로세스가 적용되었습니다.
- **`src/chatService.js`**
  - **1:1 채팅 비즈니스 로직**을 담당합니다. 닉네임을 가나다순으로 정렬한 `room-닉네임A-닉네임B` 형태의 유일 방 ID 식별 정책이 탑재되어 대화방 다중 분산 생성 버그를 방지하고 실시간 알림음 및 팝업을 완벽 연동합니다.
- **`src/supabaseClient.js`**
  - 데이터 저장소와 통신을 담당합니다. 실제 Supabase의 **posts**, **chat_rooms**, **chat_messages**, **users**, **post_comments**, **reports** 테이블과 연동됩니다. **특히 v4.3에서는 사용자의 Supabase DB 인스턴스에 `reports` 및 `post_comments` 테이블 스키마가 없는 경우에도 에러를 뱉고 멈추지 않고, 자동으로 로컬스토리지 `mockDB` 폴백(Fallback)으로 우회하도록 철저히 고안되었습니다.**
- **`src/index.css`**
  - 앱 전체의 스타일 시트입니다. 인게임 감성의 묵직한 다크 그레이와 네온 연두/노랑 테마 스타일링이 전격 설계되어 글래스모피즘 분위기를 물씬 줍니다.
- **`src/stickersData.js`**
  - 위하이브 동적 크래핑을 통해 빌드된 180장의 정규 스티커 초고화질 CDN 주소 목록과 한글 정명 매핑 로직이 들어있습니다.
- **`supabase_schema.sql`**
  - Supabase 모드 운영 시 필요한 신규 테이블 DDL 생성 쿼리 스펙 파일입니다.
- **`scripts/` (데이터 수집 및 크롭 도구)**
  - `scraper.js`: Ulixee Hero 크롤러를 띄워 위하이브 공식 글에서 180장의 실물 이미지 CDN 주소를 수집합니다.
  - `crop_thumbnails.py`: Pillow 이미지 라이브러리를 통해 20개 카테고리의 1번 이미지 배지 영역을 크롭하여 `public/sticker_images/`에 저장합니다.

## ⚡ 교환 매칭 & 실시간 1:1 채팅의 원리 (v4.3)

1. **로그인 및 회원 정보 기억 시스템**:
   - `💾 로그인 & 회원가입 (자동 저장)`: 닉네임과 **비밀번호**를 한 세트로 등록하여 로그인합니다.
   - `⚡ 일회용 게스트로 시작하기`: 닉네임만 적어 SessionStorage에 임시 저장합니다. 이때 게스트 계정의 닉네임 상태는 항상 `닉네임 (게스트)` 형태로 통일되어 할당 및 저장됩니다.
2. **양방향 복합 바구니**:
   - 한 사용자가 줄 수 있는 카드(Haves)와 받고 싶은 카드(Wants)를 동시에 바구니에 담아 글 하나로 등록합니다.
3. **매칭 알고리즘**:
   - **완벽 매칭 (100% 매칭)**: 내가 줄 수 있는 카드와 상대방이 원하는 카드가 겹치고, 동시에 내가 원하는 카드와 상대방이 줄 수 있는 카드가 겹치는 경우 (`⚡ 100% 매칭 완료!` 배지).
   - **부분 매칭**: 어느 한쪽 방향으로만 교환이 매칭되는 경우 (`💡 부분 매칭` 배지).
4. **실시간 1:1 채팅 (수발신 통합)**:
   - **수신/발신 통합**: 두 대화 당사자의 닉네임을 가나다순으로 자동 정렬하여 하나의 고유 방 ID(`room-닉네임A-닉네임B`)를 생성해 매핑하므로, 여러 글에서 동일인에게 말을 걸더라도 방이 중구난방 쪼개지지 않고 **단 하나의 대화방**으로 결합되어 메시지를 수발신합니다.
5. **실시간 Presence & 소켓 채팅 연동**:
   - Supabase Presence API 및 로컬스토리지 Heartbeat 폴백 시스템을 통하여, 로그인 중인 유저들의 접속 상태를 실시간 동기화합니다.
6. **글 수정 / 댓글 / 신고 및 관리 권한 체계 & DB 폴백**:
   - **교환글 수정**: 내가 올린 글의 보유/필요 스티커 목록과 연락처 정보를 실시간으로 수정하며, 미니 스티커 도감 인터페이스를 모달 안에 내장해 모바일/PC 모두에서 직관적으로 수정이 가능합니다.
   - **댓글 아코디언 및 신고 접수**: 거래글 하단 댓글 및 부적절한 게시글/댓글 신고 기능이 제공됩니다. **만약 Supabase 상에 `post_comments` 혹은 `reports` 테이블이 만들어져 있지 않아 `relation does not exist` 에러가 감지되는 환경일 경우, 브라우저 콘솔에 경고를 남기고 로컬스토리지 모드로 즉시 우회(Fallback) 처리하여 안전하게 동작시킵니다.**
   - **관리자 전용 권한**: 로그인 닉네임이 `간장`일 때만 관리자 권한이 부여되며, 누적된 신고 목록을 통합 모니터링하여 대상 게시글/댓글을 원클릭으로 강제 삭제 및 조치 종결할 수 있습니다.

## 🗓️ 업데이트 기록 (2026-06-06)

- **v3.0** ~ **v4.1**: 일회용 로그인 및 비밀번호 기반 로그인, 양방향 매칭 엔진, 실시간 1:1 채팅, 띵동 알림음, 안 읽은 개수 배지, 우하단 토스트 알림 팝업, 내 정보 사이드 드로어 UI 개발, 연락처 선택 입력화, 3x3 도감 페이지 전환 화살표 내비게이션, 온라인/오프라인 접속 유저 실시간 Presence 등.
- **v4.2 수신/발신 1:1 통합 채팅방 구현 및 실시간 알림 복구**:
  - 참여자 닉네임을 가나다순으로 자동 정렬하여 단 하나의 고유 방 ID(`room-닉네임A-닉네임B`)를 생성해 통합 대화방 개설.
  - 글 ID 및 누가 먼저 말을 걸었는지와 무관하게 무조건 단 하나의 1:1 채팅방으로 매핑되도록 처리하여 창 다중 분산 생성 차단.
- **v4.3 댓글/신고 테이블 부재 대비 자동 로컬 폴백 탑재 [LATEST]**:
  - Supabase 데이터베이스 상에 `post_comments` 및 `reports` 테이블이 생성되어 있지 않은 환경(schema cache table missing)에서 신고 제출 및 댓글 작성 시 에러 경고창이 뜨며 기능이 정지되는 오동작 해결.
  - 관련 DB API(fetch/add/remove/resolve)에 try-catch 예외 처리 및 테이블 부재 오류 판별 조건식을 탑재하여, 에러 발생 시 자동으로 브라우저 LocalStorage 백업 저장소(`mockDB`)로 경로를 우회시킴으로써 100% 끊김 없는 서비스 제공.
  - 추후 사용자가 Supabase SQL Editor에 우리가 작성해 둔 DDL을 실행하여 테이블을 정상 생성하는 즉시 원격 DB 모드로 코렉션 없이 연결됨.
