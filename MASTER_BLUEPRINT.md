## 📂 파일 구조 설명

이 프로젝트의 핵심 파일들은 다음과 같이 배치되어 있습니다:

- **`index.html`**
  - 사이트의 뼈대가 되는 파일입니다. 검색엔진 노출을 위한 정보(SEO)와 한국어 설정이 적용되어 있습니다.
- **`src/main.jsx`**
  - React 앱의 진입점으로, `App.jsx`와 `index.css`를 연결해 브라우저에 화면을 띄워주는 역할을 합니다.
- **`src/App.jsx`**
  - **애플리케이션의 핵심**입니다. 닉네임+비밀번호 기반 로그인/회원가입 제어, 복합 선택 바구니(Haves/Wants), 3x3 스티커 도감, 실시간 양방향 매칭 엔진, 그리고 1:1 채팅창 UI 및 액션 로직이 모두 집약되어 있습니다. 특히 상세 도감 화면 상단에 바로 옆 카테고리로 1초 만에 스위칭할 수 있는 꺾쇠 내비게이션(`<`, `>`)이 내장되어 있습니다. 또한 온라인 접속 유저 목록(`onlineUsers`)을 닉네임 옆과 채팅창 헤더에 인디케이터 배지(🟢 온라인 / ⚪ 오프라인)로 실시간으로 렌더링해 줍니다.
- **`src/chatService.js`**
  - **1:1 채팅 비즈니스 로직**을 담당합니다. 로컬스토리지 모드 및 Supabase 모드 모두 지원하며, `window.dispatchEvent`와 `storage` 이벤트 리스너를 활용해 서로 다른 브라우저 탭/창 간에 실시간 1:1 양방향 통신을 보장합니다.
- **`src/index.css`**
  - 앱 전체의 스타일 시트입니다. 인게임 감성의 묵직한 다크 그레이와 네온 연두/노랑 테마 스타일링이 전격 설계되어 글래스모피즘 분위기를 물씬 줍니다.
- **`src/stickersData.js`**
  - 위하이브 동적 크래핑을 통해 빌드된 **180장의 정규 스티커 초고화질 CDN 주소 목록**과 한글 정명 매핑 로직이 들어있습니다.
- **`src/supabaseClient.js`**
  - 데이터 저장소와 통신을 담당합니다. 데이터베이스 연결이 완료되었을 때 실제 Supabase의 **posts**, **chat_rooms**, **chat_messages**, **users** 테이블과 연동되어 실시간 대시보드를 생성하며, 백업 및 로컬 가동 시에는 LocalStorage 백업 엔진으로 자연스럽게 스위칭됩니다.
- **`scripts/` (데이터 수집 및 크롭 도구)**
  - `scraper.js`: Ulixee Hero 크롤러를 띄워 위하이브 공식 글에서 180장의 실물 이미지 CDN 주소를 수집합니다.
  - `crop_thumbnails.py`: Pillow 이미지 라이브러리를 통해 20개 카테고리의 1번 이미지 배지 영역을 크롭하여 `public/sticker_images/`에 저장합니다.

## ⚡ 교환 매칭 & 실시간 1:1 채팅의 원리 (v3.5)

1. **로그인 및 회원 정보 기억 시스템**:
   - `💾 로그인 & 회원가입 (자동 저장)`: 닉네임과 **비밀번호**를 한 세트로 등록하여 로그인합니다. 최초 로그인 시 자동으로 계정이 가입 등록(`users` 테이블)되며, 이후 로그인 시 비밀번호가 다르면 진입을 차단합니다. 로컬 세션 자격 증명(Credentials)을 세트로 보존 및 기억합니다.
   - `⚡ 일회용 게스트로 시작하기`: 닉네임만 적어 SessionStorage에 임시 저장하며, 브라우저 탭을 닫으면 자동으로 소멸됩니다.
2. **양방향 복합 바구니**:
   - 한 사용자가 줄 수 있는 카드(Haves)와 받고 싶은 카드(Wants)를 동시에 바구니에 담아 글 하나로 등록합니다.
3. **매칭 알고리즘**:
   - **완벽 매칭 (100% 매칭)**: 내가 줄 수 있는 카드와 상대방이 원하는 카드가 겹치고, 동시에 내가 원하는 카드와 상대방이 줄 수 있는 카드가 겹치는 경우 (`⚡ 100% 매칭 완료!` 배지).
   - **부분 매칭**: 어느 한쪽 방향으로만 교환이 매칭되는 경우 (`💡 부분 매칭` 배지).
4. **실시간 1:1 채팅**:
   - 피드에서 다른 사람의 글에 있는 `1:1 채팅`을 누르면 즉시 1:1 대화방이 생성되고, 우하단 채팅 팝업에서 대화가 시작됩니다.
   - 로컬스토리지 모드에서는 브라우저의 Storage 이벤트를 Listen하여, 탭을 두 개 열어두고 대화할 때 실시간으로 메시지가 전송 및 수신됩니다.
5. **실시간 Presence & 소켓 채팅 연동 (v3.5)**:
   - **Supabase Realtime 메시징**: Supabase 모드가 활성화되면 `chat_messages` 테이블의 `postgres_changes` 이벤트를 실시간으로 구독(Subscribe)하여 새로운 메시지가 추가되는 즉시 대화 창에 수신되어 출력됩니다.
   - **실시간 접속 여부 표시**: Supabase Presence API 및 로컬스토리지 Heartbeat 폴백 시스템을 통하여, 로그인 중인 유저들의 접속 상태를 실시간 동기화합니다. 피드 닉네임 옆 및 채팅 헤더에서 실시간 접속 여부(🟢 온라인 / ⚪ 오프라인) 상태를 감지할 수 있습니다.
   - **원격 자동 구성 완료**: 에이전트 자동 스크립트를 통해 Supabase SQL DB와 직접 통신하여 `chat_messages` 및 `users` 테이블에 대한 실시간 수신 구독 권한 설정(ALTER PUBLICATION)을 직접 인가 완료했습니다.
