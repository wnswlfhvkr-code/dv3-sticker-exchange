## 📂 파일 구조 설명

이 프로젝트의 핵심 파일들은 다음과 같이 배치되어 있습니다:

- **`index.html`**
  - 사이트의 뼈대가 되는 파일입니다. 검색엔진 노출을 위한 정보(SEO)와 한국어 설정이 적용되어 있습니다.
- **`src/main.jsx`**
  - React 앱의 진입점으로, `App.jsx`와 `index.css`를 연결해 브라우저에 화면을 띄워주는 역할을 합니다.
- **`src/App.jsx`**
  - **애플리케이션의 핵심**입니다. 닉네임+비밀번호 기반 로그인/회원가입 제어, 복합 선택 바구니(Haves/Wants), 3x3 스티커 도감, 실시간 양방향 매칭 엔진, 그리고 1:1 채팅창 UI 및 액션 로직이 모두 집약되어 있습니다. 상세 도감 화면 상단에 바로 옆 카테고리로 1초 만에 스위칭할 수 있는 꺾쇠 내비게이션(`<`, `>`)이 내장되어 있습니다. 또한 온라인 접속 유저 목록(`onlineUsers`)을 실시간으로 렌더링해 줍니다. v4.8에서는 **황금테두리 교환 격리 차단**, **도감 클릭 시 등록글 실시간 자동수정 연동(`updatePostStickersDirectly`)**, **피드 스티커 목록 아코디언 토글형 레이아웃**, **글 수정 모달 3x3 격자도감화**, **마우스 좌클릭(Haves)/우클릭(Wants) 카드 입력 통합**, **키보드 '4'/'6' 단축키 바인딩**이 구현되어 최상의 모바일 및 PC 인터랙션 편의성을 제공합니다.
- **`src/chatService.js`**
  - **1:1 채팅 비즈니스 로직**을 담당합니다. 닉네임을 가나다순으로 정렬한 `room-닉네임A-닉네임B` 형태의 유일 방 ID 식별 정책이 탑재되어 대화방 다중 분산 생성 버그를 방지하고 실시간 알림음 및 팝업을 완벽 연동합니다. `getMyChatRooms`에서 최신 규격(3세그먼트) 검사 필터링을 통해 구버전 방 유입을 원천 배제합니다.
- **`src/supabaseClient.js`**
  - 데이터 저장소와 통신을 담당합니다. 실제 Supabase의 **posts**, **chat_rooms**, **chat_messages**, **users**, **post_comments**, **reports** 테이블과 연동됩니다. 특히 사용자의 Supabase DB 인스턴스에 `reports` 및 `post_comments` 테이블 스키마가 없는 경우에도 에러를 뱉고 멈추지 않고, 자동으로 로컬스토리지 `mockDB` 폴백(Fallback)으로 우회하도록 철저히 고안되었습니다.
- **`src/index.css`**
  - 앱 전체의 스타일 시트입니다. 인게임 감성의 묵직한 다크 그레이와 네온 연두/노랑 테마 스타일링이 전격 설계되어 글래스모피즘 분위기를 물씬 줍니다.
- **`src/stickersData.js`**
  - 위하이브 동적 크래핑을 통해 빌드된 180장의 정규 스티커 초고화질 CDN 주소 목록과 한글 정명 매핑 로직이 들어있습니다.
- **`supabase_schema.sql`**
  - Supabase 모드 운영 시 필요한 신규 테이블 DDL 생성 쿼리 스펙 파일입니다.
- **`scripts/` (데이터 수집 및 크롭 도구)**
  - `scraper.js`: Ulixee Hero 크롤러를 띄워 위하이브 공식 글에서 180장의 실물 이미지 CDN 주소를 수집합니다.
  - `crop_thumbnails.py`: Pillow 이미지 라이브러리를 통해 20개 카테고리의 1번 이미지 배지 영역을 크롭하여 `public/sticker_images/`에 저장합니다.

## ⚡ 교환 매칭 & 실시간 1:1 채팅의 원리 (v4.8)

1. **로그인 및 회원 정보 기억 시스템**:
   - `💾 로그인 & 회원가입 (자동 저장)`: 닉네임과 **비밀번호**를 한 세트로 등록하여 로그인합니다.
   - `⚡ 일회용 게스트로 시작하기`: 닉네임만 적어 SessionStorage에 임시 저장합니다. 이때 게스트 계정의 닉네임 상태는 항상 `닉네임 (게스트)` 형태로 통일되어 할당 및 저장됩니다.
2. **양방향 복합 바구니**:
   - 한 사용자가 줄 수 있는 카드(Haves)와 받고 싶은 카드(Wants)를 동시에 바구니에 담아 글 하나로 등록합니다.
3. **매칭 알고리즘**:
   - **완벽 매칭 (100% 매칭)**: 내가 줄 수 있는 카드와 상대방이 원하는 카드가 겹치고, 동시에 내가 원하는 카드와 상대방이 줄 수 있는 카드가 겹치는 경우 (`⚡ 100% 매칭 완료!` 배지).
   - **부분 매칭**: 어느 한쪽 방향으로만 교환이 매칭되는 경우 (`💡 부분 매칭` 배지).
4. **실시간 1:1 채팅 및 목록 동기화**:
   - **수신/발신 통합**: 두 대화 당사자의 닉네임을 가나다순으로 자동 정렬하여 하나의 고유 방 ID(`room-닉네임A-닉네임B`)를 생성해 매핑하므로, 여러 글에서 동일인에게 말을 걸더라도 방이 중구난방 쪼개지지 않고 **단 하나의 대화방**으로 결합되어 메시지를 수발신합니다.
   - **실시간 대화 목록 업데이트**: 내가 메시지를 보내거나 상대방이 실시간으로 메시지를 전송할 때, 글로벌 메시지 리스너(`subscribeAllMyMessages`) 및 로컬 탭 동기화 리스너(`dv3_chat_update`)가 즉각 `loadChatRooms()`를 호출하여 목록을 항상 최신 메시지와 시각 기준으로 갱신합니다.
   - **목록 하이라이팅**: 채팅방 목록 상에서 현재 사용자가 클릭하여 활성화한 대화방(`room.id === chatActiveRoomId`)에 연두색 네온 테두리와 투명 네온 광채 하이라이팅 효과를 부여하여, 사용자 경험의 매칭성을 완벽하게 제공합니다.
5. **실시간 Presence & 소켓 채팅 연동**:
   - Supabase Presence API 및 로컬스토리지 Heartbeat 폴백 시스템을 통하여, 로그인 중인 유저들의 접속 상태를 실시간 동기화합니다.
6. **자동 데이터 마이그레이션**:
   - 사용자가 로그인 시, `runDatabaseMigration` 함수가 작동하여 DB에 저장되어 있던 모든 구버전 분할 대화방 레코드를 자동 스캔합니다.
   - 해당 구버전 대화방에 존재하던 대화 메시지들을 최신 1:1 통합 대화방 ID(`room-닉네임A-닉네임B`)로 일괄 이관(Update)하고, 기존 구버전 방 레코드는 영구 삭제(Delete) 처리하여 한 대화 당사자 간에는 **결국 단 하나의 대화방만 존재**하게 데이터를 완벽 정화합니다.
7. **글 스티커 정보와 상단 바구니 간의 양방향 자동 동기화**:
   - 사용자가 내 교환글을 '수정'하면, 완료 즉시 수정한 카드 데이터(`editHaves`, `editWants`)가 화면 상단 나의 바구니(`myHaves`, `myWants`)에 직접 반영됩니다.
   - 또한 백그라운드 데이터 폴링 발생 시에도 `syncMyBasketFromPost` 모듈이 최근 게시글 정보를 대조하여, 나의 글 정보가 바뀌면 바구니 상태도 자동 연동 업데이트해 줍니다. 이로써 수정 완료 후 매칭 계산 및 뱃지 출력이 딜레이 없이 100% 실시간 동적 재계산됩니다.
8. **바구니 스티커 제거(X) 시 상호 바구니 추가 차단**:
   - Wants 나 Haves 바구니에서 임의의 스티커를 뺄 때, 선택 중인 모드와 관련 없이 해당 바구니에서만 깔끔히 빠지도록 `targetMode` 파라미터 제어를 이식하여 오동작을 원천 차단했습니다.
9. **도감 마우스 좌/우클릭 통합 입력 및 즉시 수정 연동 (v4.8)**:
   - 메인 도감 뷰 내의 개별 스티커 카드를 **마우스 좌클릭 시 보유 카드(Haves), 우클릭 시 필요 카드(Wants)**에 즉시 등록/해제할 수 있도록 입력을 단일 창에 통합했습니다.
   - 내 등록 글이 이미 생성된 상태라면, 도감 카드를 누르는 즉시 `updatePostStickersDirectly` 헬퍼를 통해 Supabase 원격 DB와 로컬 피드 데이터가 백그라운드에서 지연 없이 실시간 수정 및 동기화됩니다.
10. **황금테두리(isGolden) 교환 격리 차단 (v4.8)**:
    - 인게임 상 교환불가 스펙인 황금테두리 카드는 도감 내에 🔒 자물쇠 배지와 `"교환 불가"` 어두운 필터링 레이아웃이 적용되며 클릭/선택이 완전 차단됩니다. 원격 DB 백그라운드 동기화 단계에서도 황금테두리 카드는 바구니 연동에서 전격 필터링 배제 처리됩니다.

## 🗓️ 업데이트 기록 (2026-06-06)

- **v3.0** ~ **v4.2**: 일회용 로그인 및 비밀번호 기반 로그인, 양방향 매칭 엔진, 실시간 1:1 채팅, 띵동 알림음, 안 읽은 개수 배지, 우하단 토스트 알림 팝업, 내 정보 사이드 드로어 UI 개발, 연락처 선택 입력화, 3x3 도감 페이지 전환 화살표 내비게이션, 온라인/오프라인 접속 유저 실시간 Presence 등.
- **v4.3 댓글/신고 테이블 부재 대비 자동 로컬 폴백 탑재**:
  - Supabase 데이터베이스 상에 `post_comments` 및 `reports` 테이블이 없는 경우에도 에러를 뿜으며 멈추지 않고, 자동으로 로컬스토리지 `mockDB` 폴백으로 우회해 댓글/신고를 가능하게 함.
- **v4.4 관리자 신고 로그 추가 및 1:1 채팅방 목록 실시간 매칭 동기화**:
  - 관리자가 신고 내역을 반려하거나 대상(글/댓글)을 강제 삭제했을 때 조치 내역이 사라지지 않도록 관리자 대시보드 내에 **조치 완료 로그** 탭을 추가하고 로컬스토리지(`dv3_admin_resolved_logs`)에 실시간 영구 보존함.

  - **1:1 채팅 비즈니스 로직**을 담당합니다. 닉네임을 가나다순으로 정렬한 `room-닉네임A-닉네임B` 형태의 유일 방 ID 식별 정책이 탑재되어 대화방 다중 분산 생성 버그를 방지하고 실시간 알림음 및 팝업을 완벽 연동합니다. `getMyChatRooms`에서 최신 규격(3세그먼트) 검사 필터링을 통해 구버전 방 유입을 원천 배제합니다.
- **`src/supabaseClient.js`**
  - 데이터 저장소와 통신을 담당합니다. 실제 Supabase의 **posts**, **chat_rooms**, **chat_messages**, **users**, **post_comments**, **reports** 테이블과 연동됩니다. 특히 사용자의 Supabase DB 인스턴스에 `reports` 및 `post_comments` 테이블 스키마가 없는 경우에도 에러를 뱉고 멈추지 않고, 자동으로 로컬스토리지 `mockDB` 폴백(Fallback)으로 우회하도록 철저히 고안되었습니다.

## ⚡ 교환 매칭 & 실시간 1:1 채팅의 원리 (v4.7)

1. **로그인 및 회원 정보 기억 시스템**:
   - `💾 로그인 & 회원가입 (자동 저장)`: 닉네임과 **비밀번호**를 한 세트로 등록하여 로그인합니다.
   - `⚡ 일회용 게스트로 시작하기`: 닉네임만 적어 SessionStorage에 임시 저장합니다. 이때 게스트 계정의 닉네임 상태는 항상 `닉네임 (게스트)` 형태로 통일되어 할당 및 저장됩니다.
2. **양방향 복합 바구니**:
   - 한 사용자가 줄 수 있는 카드(Haves)와 받고 싶은 카드(Wants)를 동시에 바구니에 담아 글 하나로 등록합니다.
3. **매칭 알고리즘**:
   - **완벽 매칭 (100% 매칭)**: 내가 줄 수 있는 카드와 상대방이 원하는 카드가 겹치고, 동시에 내가 원하는 카드와 상대방이 줄 수 있는 카드가 겹치는 경우 (`⚡ 100% 매칭 완료!` 배지).
   - **부분 매칭**: 어느 한쪽 방향으로만 교환이 매칭되는 경우 (`💡 부분 매칭` 배지).
4. **실시간 1:1 채팅 및 목록 동기화**:
   - **수신/발신 통합**: 두 대화 당사자의 닉네임을 가나다순으로 자동 정렬하여 하나의 고유 방 ID(`room-닉네임A-닉네임B`)를 생성해 매핑하므로, 여러 글에서 동일인에게 말을 걸더라도 방이 중구난방 쪼개지지 않고 **단 하나의 대화방**으로 결합되어 메시지를 수발신합니다.
   - **실시간 대화 목록 업데이트**: 내가 메시지를 보내거나 상대방이 실시간으로 메시지를 전송할 때, 글로벌 메시지 리스너(`subscribeAllMyMessages`) 및 로컬 탭 동기화 리스너(`dv3_chat_update`)가 즉각 `loadChatRooms()`를 호출하여 목록을 항상 최신 메시지와 시각 기준으로 갱신합니다.
   - **목록 하이라이팅**: 채팅방 목록 상에서 현재 사용자가 클릭하여 활성화한 대화방(`room.id === chatActiveRoomId`)에 연두색 네온 테두리와 투명 네온 광채 하이라이팅 효과를 부여하여, 사용자 경험의 매칭성을 완벽하게 제공합니다.
5. **실시간 Presence & 소켓 채팅 연동**:
   - Supabase Presence API 및 로컬스토리지 Heartbeat 폴백 시스템을 통하여, 로그인 중인 유저들의 접속 상태를 실시간 동기화합니다.
6. **자동 데이터 마이그레이션**:
   - 사용자가 로그인 시, `runDatabaseMigration` 함수가 작동하여 DB에 저장되어 있던 모든 구버전 분할 대화방 레코드를 자동 스캔합니다.
   - 해당 구버전 대화방에 존재하던 대화 메시지들을 최신 1:1 통합 대화방 ID(`room-닉네임A-닉네임B`)로 일괄 이관(Update)하고, 기존 구버전 방 레코드는 영구 삭제(Delete) 처리하여 한 대화 당사자 간에는 **결국 단 하나의 대화방만 존재**하게 데이터를 완벽 정화합니다.
7. **글 스티커 정보와 상단 바구니 간의 양방향 자동 동기화**:
   - 사용자가 내 교환글을 '수정'하면, 완료 즉시 수정한 카드 데이터(`editHaves`, `editWants`)가 화면 상단 나의 바구니(`myHaves`, `myWants`)에 직접 반영됩니다.
   - 또한 백그라운드 데이터 폴링 발생 시에도 `syncMyBasketFromPost` 모듈이 최근 게시글 정보를 대조하여, 나의 글 정보가 바뀌면 바구니 상태도 자동 연동 업데이트해 줍니다. 이로써 수정 완료 후 매칭 계산 및 뱃지 출력이 딜레이 없이 100% 실시간 동적 재계산됩니다.
8. **바구니 스티커 제거(X) 시 상호 바구니 추가 차단**:
   - Wants 나 Haves 바구니에서 임의의 스티커를 뺄 때, 선택 중인 모드와 관련 없이 해당 바구니에서만 깔끔히 빠지도록 `targetMode` 파라미터 제어를 이식하여 오동작을 원천 차단했습니다.
