# AdSense Renewal Goals

작성일: 2026-07-09
작업 브랜치: `codex/adsense-renewal-goals`
대상 사이트: `https://dv3-sticker.vercel.app/`

## 1. 분석 범위

- 로컬 React/Vite 앱, `public/*.html` 정적 페이지, `robots.txt`, `sitemap.xml`, `ads.txt`, 광고 관련 코드, 푸터/상단 링크 구조를 확인했다.
- 배포 URL 기준으로 홈, `ads.txt`, `sitemap.xml`, 일부 링크 상태를 확인했다.
- AdSense 기준은 Google 공식 문서 기준으로 정리했다.
  - `https://support.google.com/adsense/answer/7299563`
  - `https://support.google.com/adsense/answer/48182`
  - `https://support.google.com/adsense/answer/10502938`
  - `https://support.google.com/adsense/answer/1346295`

## 2. 현재 사이트 강점

- `privacy.html`, `terms.html`, `about.html`, `guide.html`, `faq.html`이 있고 메인 푸터에서 접근 가능하다.
- `tips-gem-reinforcement.html`, `tips-sticker-collection.html`, `tips-safe-trading.html`, `tips-matching-guide.html`처럼 검색 가능한 정적 정보 페이지가 있다.
- `robots.txt`, `sitemap.xml`, `ads.txt`가 배포되어 있다.
- `AdBanner` 컴포넌트는 현재 수동 광고 슬롯을 렌더링하지 않아, 버튼 주변에 임의 광고 슬롯이 섞이는 구조는 아직 없다.
- 이용규칙, 신고 모달, 관리자 대시보드 등 사용자 생성 콘텐츠 관리를 위한 기본 구조가 있다.

## 3. AdSense 관점 주요 리스크

### 3.1 첫 방문 콘텐츠 접근성

현재 비로그인 사용자는 `LoginModal`이 닫을 수 없는 오버레이로 먼저 노출된다. 본문 DOM은 렌더링되더라도 시각적으로는 로그인 게이트가 콘텐츠를 막는다. Google의 "고유 콘텐츠와 좋은 사용자 경험", "명확하고 쉬운 내비게이션" 기준에서 불리하다.

### 3.2 메인 화면의 콘텐츠 가치 인상

메인은 교환 도구, 사용자 게시글, 채팅, 신고 버튼 중심이다. 정보성 정적 페이지는 존재하지만 메인 첫 화면에서는 콘텐츠 사이트보다 실시간 거래/커뮤니티 도구처럼 보인다. AdSense의 낮은 가치 콘텐츠, 비콘텐츠 페이지, 사용자 행동 목적 페이지 리스크를 줄이려면 메인에도 검토 가능한 독자 콘텐츠가 필요하다.

### 3.3 깨진 링크

배포 URL에서 `/about-stickers.html`은 404다. `tips-sticker-collection.html`, `tips-safe-trading.html`, `tips-matching-guide.html`의 상단/하단 내비게이션이 이 경로를 참조한다. 이는 Site behavior / Navigation 리스크다.

### 3.4 내비게이션 버그

`Header`는 로고 클릭 시 `setSelectedCategoryId(null)`을 호출하지만 `App.jsx`에서 해당 prop을 전달하지 않는다. 실제 클릭 시 런타임 오류가 날 수 있어 내비게이션 안정성 리스크다.

### 3.5 사용자 생성 콘텐츠 관리 고지 부족

교환글, 댓글, 자유게시판, 1:1 채팅이 핵심 기능이다. 이용규칙은 있지만 메인에서 금지 행위, 신고/삭제 기준, 현금거래 금지, 사기 대응, 부적절 콘텐츠 제재가 충분히 드러나지 않는다. AdSense 정책상 댓글/UGC는 운영자가 관리해야 하며 부적절 콘텐츠 누적은 심사와 광고 게재에 불리하다.

### 3.6 저작권/상표권 리스크

드래곤빌리지3 명칭과 스티커 이미지는 제3자 IP다. 푸터에 비공식 도구와 권리 귀속 문구가 있으나, 광고 수익화 관점에서는 "복제 콘텐츠" 또는 "저작권 침해"로 보이지 않도록 원본 해설, 공략, 자체 분석, 비공식 팬 도구 고지가 더 명확해야 한다. 특히 게임 이미지 위주 화면에는 광고 배치를 신중히 제한해야 한다.

### 3.7 개인정보처리방침 정확성

현재 개인정보처리방침은 AdSense 쿠키를 언급하지만, 실제 서비스의 로컬스토리지, Supabase, 로그인 비밀번호, 닉네임, 교환 연락처, 채팅/댓글/신고 정보 처리와 완전히 맞물리지는 않는다. Google 제품 사용에 따른 쿠키/식별자 고지와 실제 수집 항목을 더 정확히 맞춰야 한다.

### 3.8 광고 배치 리스크

Google 광고는 메뉴, 버튼, 채팅, 신고, 댓글 입력, 연락하기 버튼처럼 조작 요소와 붙으면 우발 클릭 리스크가 있다. 또한 1:1 채팅처럼 private communication이 주된 화면에는 광고를 두지 않아야 한다. 현재 `index.html`과 정적 페이지에 AdSense 스크립트는 있으므로, 승인 후 수동 슬롯을 넣을 때 배치 기준을 먼저 확정해야 한다.

## 4. 리뉴얼 목표

### 목표 1. 로그인 없이 읽고 탐색 가능한 첫 화면

- 로그인 모달을 첫 진입 오버레이에서 제거하거나 닫을 수 있는 선택형 진입으로 바꾼다.
- 글쓰기, 채팅, 댓글 작성처럼 계정이 필요한 행동에서만 로그인 요청이 뜨게 한다.
- 비로그인 상태에서도 스티커북, 교환글 목록, 가이드, FAQ, 정책 페이지를 읽을 수 있게 한다.

### 목표 2. 메인에 검토 가능한 자체 콘텐츠 추가

- 메인 상단에 "이 사이트가 무엇을 해결하는지", "스티커 교환을 안전하게 하는 방법", "자동 매칭 원리", "운영/신고 원칙"을 짧지만 실질적인 본문으로 추가한다.
- 단순 소개 문구가 아니라 정적 공략 페이지와 연결되는 요약 콘텐츠를 배치한다.
- 첫 화면은 도구 사용성은 유지하되, 심사 봇과 신규 방문자가 사이트의 가치와 운영 목적을 즉시 이해하게 만든다.

### 목표 3. 내비게이션 품질 정리

- `/about-stickers.html`을 실제 페이지로 만들거나 모든 참조를 존재하는 페이지로 교체한다.
- 메인, 정적 페이지, 푸터의 메뉴 구조를 통일한다.
- Header 로고 클릭 오류를 수정한다.
- 모든 내부 링크가 200 응답을 반환하는지 점검한다.

### 목표 4. UGC 안전 운영을 메인 UX에 포함

- 현금거래, 계정거래, 욕설, 성인/도박/폭력 콘텐츠, 사기 유도, 광고성 도배 금지를 메인과 작성 UI 주변에 명확히 안내한다.
- 게시글/댓글 작성 전에 짧은 커뮤니티 기준을 노출한다.
- 신고/관리 정책 페이지를 보강하고 메인에서 쉽게 접근하게 한다.
- 가능하면 금칙어/위험어 사전 기반의 작성 전 경고를 추가한다.

### 목표 5. 저작권/상표권 오해 줄이기

- 메인 상단과 푸터에 "비공식 팬 제작 교환 도구"라는 성격을 더 명확히 표시한다.
- 정적 정보 페이지는 이미지 복제보다 자체 해설, 계산, 공략, 안전 가이드 중심으로 보강한다.
- 광고는 원 저작권 이미지가 주가 되는 영역보다 자체 텍스트 콘텐츠 주변에 우선 배치한다.

### 목표 6. 개인정보/광고 고지 정합성 강화

- 개인정보처리방침에 실제 수집/저장 항목을 정확히 반영한다: 닉네임, 로그인 정보, 교환 연락처, 게시글, 댓글, 신고, 채팅, 로컬스토리지, Supabase, 방문 로그.
- Google 광고/분석 제품이 쿠키, 웹 비콘, IP 주소 또는 식별자를 사용할 수 있음을 명확히 고지한다.
- Google의 "파트너 사이트 또는 앱에서 Google이 데이터를 사용하는 방식" 안내 링크를 포함한다.
- 문의 이메일과 삭제/수정 요청 절차를 더 명확히 한다.

### 목표 7. 광고 배치는 승인 후에도 보수적으로 운영

- 승인 전에는 검토용 스크립트 외에 눈에 띄는 광고 placeholder나 클릭 유도 문구를 넣지 않는다.
- 승인 후 1차 배치는 정적 정보 페이지 본문 중간/하단 또는 교환글 목록과 게시판 사이의 반응형 배너 1개로 제한한다.
- 채팅창, 연락하기 버튼, 댓글 입력창, 신고 버튼, 페이지네이션, 스티커 선택 그리드 주변에는 광고를 두지 않는다.
- 광고 라벨은 필요한 경우 "광고" 또는 "Sponsored Links" 수준으로만 사용한다.

### 목표 8. 기술 검증 기준 마련

- `npm run build`가 통과해야 한다.
- 배포 후 `robots.txt`, `sitemap.xml`, `ads.txt`, 주요 정적 페이지, 모든 내부 링크가 정상 응답해야 한다.
- 모바일 360px, 데스크톱 1280px 기준으로 첫 화면에서 모달/채팅/광고가 본문 탐색을 막지 않아야 한다.
- Google Search Console과 AdSense 재심사 전 체크리스트를 별도로 만든다.

## 5. 1차 리뉴얼 우선순위

1. 비로그인 로그인 게이트 제거 및 Header 로고 오류 수정
2. 404 내부 링크 정리 또는 `about-stickers.html` 신설
3. 메인 상단 콘텐츠/운영 원칙/비공식 고지 보강
4. 개인정보처리방침과 이용규칙 보강
5. 광고 배치 정책을 코드 구조에 반영
6. 빌드, 링크, 모바일/데스크톱 화면 검증

## 6. 이번 문서의 비목표

- 이 문서는 리뉴얼 구현 전 목표 정의 문서다.
- 실제 UI/코드 변경, 광고 슬롯 추가, 정책 페이지 문안 수정은 다음 단계에서 진행한다.
- AdSense 승인을 보장하지 않는다. 다만 Google 공식 기준과 현재 사이트 리스크를 기준으로 재심사 실패 가능성을 낮추는 방향을 정의한다.
