# 📝 드래곤 빌리지 3 카드교환소 작업 요약 (2026-06-05)

오늘 작업에서는 사용자가 메모해 둔 핵심 기능인 **일회용 로그인**, **양방향 카드 교환 등록(복합 바구니)**, **실시간 1:1 채팅 시스템**, 그리고 **닉네임+비밀번호 기반 회원 기억 시스템**을 완벽하게 구현 완료하고 런타임 검증을 마쳤습니다.

또한, **Supabase 실시간 클라우드 데이터베이스를 프로젝트에 연동하고 회원 인증 API를 동기화**했으며, **실시간 접속 상태 표시(Presence) 기능 및 실시간 채팅 수신 기능**을 완벽하게 이식했습니다.

특히, 사용자의 요청에 따라 **원격 Supabase DB에 직접 접속하여 실시간 소켓 통신을 위한 스키마 구성을 100% 원격 자동 설정**하였습니다.

이어지는 추가 작업으로, **실시간 알림음(띵동), 안 읽은 메시지 수 네온 배지 및 우하단 팝업 미리보기(Toast) 시스템**을 완전히 새롭게 장착하고 외부 서버(Vercel)에 최종 배포하였습니다.

## 🛠️ 핵심 변경 및 개선 내용

### 1. 🔔 안 읽은 메시지 수(Unread Counts) 자동 추적
- **개별 방 배지**: 대화방 리스트에서 상대방 이름 옆에 안 읽은 메시지 개수가 담긴 강렬한 레드 네온 배지가 실시간으로 증가하여 표시됩니다.
- **글로벌 토글 배지**: 우하단 초록색 둥근 말풍선 버튼 위에 현재 내가 읽지 않은 전체 메시지의 합산 수치가 배지로 노출됩니다.
- **자동 리셋 및 상태 유지**: 특정 대화방을 클릭하여 활성화하는 순간 해당 방의 안 읽은 개수는 `0`으로 자동 청소되며, 새로고침을 하거나 다른 브라우저 탭을 열어도 로컬스토리지 동기화를 통해 안 읽은 상태가 깨지지 않고 계속 보존됩니다.

### 2. 🎵 새 메시지 도착 시 띵동 오디오 알림 (Sound Notification)
- 내가 현재 채팅창을 닫아두었거나, 또는 다른 대화방을 보고 있을 때 새 메시지가 오면 청아한 띵동 알림음(Mixkit Open Source Sound)이 울려 실시간 메시지 수신을 청각적으로 알려줍니다.
- 내가 보고 있는 대화방 안에서 서로 메시지를 주고받을 때는 알림음이 울리지 않도록 세심하게 차단하여 불필요한 노이즈를 방지했습니다.

### 3. 💬 우하단 새 메시지 미리보기 토스트 (Toast Alert)
- 채팅창이 비활성화 상태이거나 다른 대화방을 보는 중에 새 메시지가 도착하면, 우하단 플로팅 버튼 바로 위에 예쁜 슬라이드업(slide-up) 토스트 알림창이 솟아오릅니다.
- 토스트 팝업창에는 보낸 사람의 닉네임과 첫 부분 내용이 미리 노출되며, 4초 후 자동으로 스무스하게 소멸합니다.
- 사용자가 이 미리보기 토스트 팝업을 클릭하면, **해당 대화방으로 순간이동(자동 포커스)**하여 즉시 답장할 수 있는 아주 뛰어난 사용성을 제공합니다.

### 4. ☁️ Supabase 원격 자동 설정 및 환경변수 동기화 완료
- Vercel 외부 배포 서버에 `VITE_SUPABASE_URL` 및 `VITE_SUPABASE_ANON_KEY` 환경변수를 원격 주입하여 외부 배포 주소에서도 완벽하게 데이터가 연동되도록 조치했습니다.
- 원격 DB의 실시간 이벤트 발행 설정(`ALTER PUBLICATION`)도 에이전트가 직접 처리 완료하였습니다.

## 🛠️ v3.7 다중 UI 사용성 개선 및 내 정보 드로어 이식 완료 (2026-06-05 추가)

### 1. 📂 내 정보 및 등록 내역 사이드 드로어 (Drawer UI)
- 헤더의 내 닉네임을 누르거나 우측의 정보(`Info`) 아이콘을 누르면, 화면 우측에서 미려한 트랜지션과 함께 슬라이딩하여 열리는 독자적인 드로어 패널을 빌드했습니다.
- 로그인한 유저가 작성한 전체 교환글의 개수를 실시간 모니터링하고, 각 글의 상세 장바구니 품목을 조회하며, 그 즉시 해당 글을 파괴(삭제)할 수 있는 삭제 버튼을 단일 드로어 내에 탑재하여 완벽한 통합 내 정보 대시보드를 연출했습니다.
- 드로어 내부에서 내 연락처를 상시 업데이트하고 저장할 수 있는 폼을 설계했습니다.

### 2. 🔑 로그인 계정 변경 시 '내 글' 오인식 권한 버그 해결
- 브라우저 로컬스토리지에 글 ID를 임시 캐싱하여 권한을 식별하던 방식을 전면 개정하여, 로그인 닉네임과 개별 게시글 작성자의 닉네임을 1대1로 비교하도록 바꿨습니다.
- 이제 동일 PC/브라우저 내에서 닉네임을 바꾸어 로그인하더라도 내 글이 타인의 아이디(예: `sd`)로 노출되거나 타인의 글을 임의로 삭제할 수 없도록 권한을 물리적으로 차단했습니다.

### 3. 🖼️ 카테고리/도감 이미지 마우스 호버 시 포토샵 툴팁 수정
- 20개 카테고리 썸네일 이미지 및 3x3 카드 도감 슬롯 내의 `<img>` 태그에 각각 `title={cat.name}`, `title={sticker.name}` 속성을 지정하여 브라우저의 원치 않는 포토샵 메타데이터 툴팁 자동 노출 오류를 원천 차단했습니다.

### 4. 🚫 도감 헤더 대표 이미지 클릭 비활성화
- 도감 상세 헤더 영역 우측 동그라미 이미지의 `onClick` 주소 이동 링크를 파괴하고 마우스 포인터 스타일을 일반 형태로 수정하여 혼선을 주는 동작을 배제했습니다.

### 5. 📞 연락처(오픈채팅 주소 등) 선택 기입화 및 피드 조건부 연락하기 연동
- 교환글 신청 시 연락처 작성을 생략(선택 사항)하고도 글이 등록되도록 백엔드/프론트엔드 검증을 완화했습니다.
- 피드 목록에서는 연락처 필드가 실제로 존재할 때만 `[연락하기]` 버튼이 뜨도록 조건부 노출 로직을 적용해 완벽하게 의도를 준수했습니다.
