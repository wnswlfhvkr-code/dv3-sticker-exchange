### [v4.9 MVVM 대규모 아키텍처 리팩토링 및 모듈화 최종 완성]
* **🏛️ 거대 모놀리식 App.jsx 리팩토링**
  - *의도/현상*: 단일 App.jsx 파일에 모여있던 3,500라인 규모의 비즈니스 로직과 화면 UI 코드로 인해 유지보수 및 협업이 어려운 현상 해소.
  - *해결 조치*: 비즈니스 로직은 **5개의 전용 ViewModel 커스텀 훅**으로 완전히 이관하고, 화면은 **12개의 독립된 순수 View 컴포넌트**로 쪼개어 파일 단위로 모듈화하였습니다. 
  - `App.jsx` 파일은 단 150라인 이하로 다이어트되어 각 모듈을 연동해주는 Entry Point(컨트롤러)로 가벼워졌습니다.

* **🔄 순환 의존성(Circular Dependency) 및 TDZ 회피**
  - *의도/현상*: 바구니(Basket) 상태와 포스트(Post) 상태 간의 양방향 자동 동기화 구조 때문에 발생하는 훅 참조 에러 방지.
  - *해결 조치*: `useRef` 지연 바인딩과 ES6 Getter 패턴을 접목하여 빌드 및 런타임 단계에서 참조 순서 에러가 나지 않는 최상의 링킹 솔루션을 구현했습니다.

* **⚙️ 안정적인 실시간 폴링 및 Presence 동기화 보존**
  - *의도/현상*: 리팩토링 과정에서 실시간 채팅, 온라인Presence, 바구니 쿨다운 싱크 기능이 누수 없이 온전히 구동되도록 유지.
  - *해결 조치*: `useBasketViewModel`, `useChatViewModel` 등을 통해 Supabase와 로컬스토리지 Heartbeat가 매끄럽게 교차 작동하도록 검증을 마쳤습니다.

* **로컬 개발 서버 렌더링 검증 완료**: `npm run dev` 실행 후 브라우저 subagent를 가동하여 `http://localhost:5174/` 경로로 마운트 및 렌더링 검증을 마쳤습니다. 기존의 categories Props 누락 및 `checkMatching` 누락으로 인한 런타임 화이트아웃 크래시를 전격 해결하여, 모든 화면 기능과 도감, 피드, 댓글, 임시 게스트 로그인이 매끄럽고 안정적으로 작동하는 상태를 100% 확인했습니다.
* **v4.9.2 화면 중앙 정렬 및 렌더링 오동작/이미지 버그 검증 완료**:
  - `align-items: center` 스타일 적용으로 가로폭 800px의 주요 UI 요소들이 브라우저 중앙에 좌우 균등 대칭으로 완전 정렬되는 것을 보장했습니다.
  - 카테고리 클릭 시 기존 선택 창이 접히고 그 위에 3x3 격자도감이 표시되는 상호 배타적 교체 렌더링을 구현하여 화면 하단에 중복 창이 더 이상 뜨지 않도록 완성했습니다.
  - 이미지 속성명을 `sticker.image`로 전면 정상화하여 180장의 드래곤 스티커 실물 사진들이 고화질 CDN 링크를 통해 완벽히 출력되는 것을 브라우저 subagent 검사 및 캡처로 입증했습니다.
* **v4.9.3 도감 기획 복구 및 1:1 실시간 채팅 갱신 버그 검증 완료**:
  - **카테고리 대표 이미지**: 도감 제목 옆에 1번 스티커 이미지(`repImageUrl`)가 원형 아바타로 세련되게 렌더링되는 것을 확인했습니다.
  - **플로팅 네비게이션 복원**: `glass-card` 외부 부모인 `detail-work-layout` 래퍼에 `position: relative`와 absolute 버튼들을 배치하여, `overflow: hidden`의 잘림 제약 없이 3x3 격자 양옆에 `◀ 이전`, `▶ 다음` 플로팅 버튼이 안전하고 대칭성 있게 화면에 표출되는 것을 스크린샷으로 확인했습니다.
  - **가이드 문구 위치 상향**: 마우스 조작 안내 배지가 타이틀 바 바로 하단으로 깔끔하게 올라가 시인성이 극대화되었습니다.
  - **황금 카드 원색 표출 및 토스트 경고**: 회색 필터가 제거되어 9번 황금 카드가 밝은 금색 원색으로 렌더링되고, 마우스 클릭 시 상단에 `"🔒 골든 등급 카드는 교환할 수 없습니다!"` 경고 배지가 3초간 실시간 노출되는 로직을 검증 완료했습니다.
  - **1:1 실시간 채팅 갱신**: `subscribeMyRooms`를 적용하여 새로운 대화방 생성 시 목록에 실시간 갱신(새로고침 불필요)되는 기능을 검증했습니다.
  - **피드 카드 별 개수별 분기**: 매칭되는 카드들을 ★5성, ★4성 단위로 묶어 가독성 있게 정리하여 보여주며, 기존의 "100% 매칭 완료" 문구를 직관적인 "교환 가능"으로 수정 완료했습니다.
* **v4.9.4 도감 카드 이미지 꽉 채우기 디자인 검증 완료**:
  - **카드 이미지 Cover 피팅**: 카드 슬롯 내부 패딩을 전면 축소(padding: 0)하고 이미지의 `objectFit: 'cover'` 및 `height: '100%'` 속성을 적용함으로써, 3x3 도감 카드 슬롯 내부에 모든 드래곤 스티커 일러스트가 빈틈없이 꽉 채워지고 확대되어 노출되는 상태를 브라우저 subagent 캡처로 성공적으로 입증했습니다.
  - **컴포넌트 오버레이 배치**: 별 등급과 스티커 이름 표시 바를 각각 좌측 상단 반투명 플로팅 배지와 하단 그라데이션 레이어로 이식하여, 이미지 전체 뷰를 가리지 않고 매우 깔끔하고 고급스러운 모바일/웹 게임 감성의 인게임 디자인으로 개선했습니다.
* **v4.9.5 스티커 원본 비율 보존 및 종횡비(aspectRatio) 카드 프레임 검증 완료**:
  - **aspectRatio 3/4**: 기존의 `height: '210px'` 고정 높이를 전면 배제하고, `aspectRatio: '3 / 4'` 반응형 비율을 적용하여 기기 너비에 대응하는 세로형 TCG 카드 비율을 구현했습니다.
  - **Contain 복구를 통한 비율 무결성 보장**: `objectFit: 'contain'`을 통해 이미지 원본 비율이 늘어나거나 찌그러지지 않고 원본의 해상도 그대로 출력되며, 카드 외곽선 테두리가 이미지 테두리와 정확히 맞아떨어져 빈 여백 없이 꽉 찬 카드 프레임을 완성했음을 실시간 스크린샷으로 입증 완료했습니다.
* **v4.9.6 카테고리 3열 그리드 및 이름 내비게이션, 띄어쓰기 교정 검증 완료**:
  - **카테고리 3열 그리드**: 카테고리 선택창을 3열로 고정하고 개별 카드를 가로세로로 크게 확대하여(230px), 각 카테고리의 대표 드래곤 일러스트가 매우 시원하고 웅장하게 표출되도록 레이아웃 변경을 확인했습니다.
  - **띄어쓰기 정화**: "야생 몬스터", "화산 지대", "어둠 속 존재", "하늘 나라", "놀이 시간" 등 띄어쓰기가 적용된 스티커 및 카테고리 명칭이 메인 목록 및 바구니 칩셋에 완벽히 출력되는 상태를 확인했습니다.
  - **카테고리명 이동 내비게이션**: 상세 도감 그리드 상단 화살표의 숫자 단축키 표기를 지우고, `< 바다 아래`, `빛 아래 >`와 같이 이전/다음 카테고리명을 표시하여 직관적인 조작을 돕도록 UX 개선을 검증했습니다.
