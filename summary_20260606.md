# 📝 2026-06-06 업데이트 요약본 (v4.3)

## 1. 완료된 사용자 요청 (Accomplished Requests)

* **🛡️ 신고 및 댓글 기능 Supabase DB 테이블 누락 대비 자동 로컬 폴백(Fallback) 구현 [LATEST]**
  - 사용자가 Supabase DB 인스턴스에 `reports` 및 `post_comments` 테이블을 생성하지 않은 상태에서 신고를 제출할 때 `"Could not find the table 'public.reports' in the schema cache"` 에러 경고창이 뜨며 기능이 멈추던 문제를 완전히 방어했습니다.
  - `fetchComments`, `addComment`, `removeComment`, `addReport`, `fetchReports`, `resolveReport` 등 댓글/신고와 관련된 전체 DB API에 대해, Supabase DB 상에 테이블이 존재하지 않는 경우 자동으로 경고(`console.warn`)를 남기고 로컬스토리지(`mockDB`) 우회 프로세스로 처리되도록 안전장치를 설계했습니다.
  - 이로써 원격 DB에 테이블이 없더라도 사용자 경험이 끊기지 않고 100% 원활하게 신고 접수 및 처리가 진행됩니다.
  - 사용자가 추후 Supabase SQL Editor를 통해 테이블을 생성하면, 프론트엔드 코드 수정 없이 즉시 실시간 원격 DB 모드로 동적 연동이 이뤄집니다.
* **💬 수신/발신 1:1 통합 유일 채팅방 구현 및 다중 생성 차단**
  - 거래글마다 각각 1:1 채팅방이 쪼개져 여러 개 중복 생성되던 오류를 해결하고, 나와 상대방 닉네임 쌍(예: `A`와 `B`) 기준 **단 하나의 유일한 1:1 채팅방**만 구성되도록 통합했습니다.
  - 대화 참여자의 닉네임을 가나다순으로 자동 정렬한 고유 방 ID(`room-닉네임A-닉네임B`)를 대입하여 중복 쪼개짐을 완벽히 차단했습니다.
* **🔔 실시간 글로벌 메시지 알림 및 미리보기 팝업 완벽 복구**
  - Supabase 모드에서 1:1 통합 방 ID(`roomId`)가 닉네임 조합 텍스트 형태로 데이터베이스에 지정 생성됨에 따라, `chat_messages` 테이블의 `room_id.includes(myNickname)` 글로벌 실시간 구독 조건이 100% 참(true)으로 충족됩니다.
  - **청아한 띵동 알림음 재생**, **우하단 실시간 토스트 메시지 미리보기 팝업**, **안 읽은 개수 네온 배지(● N)** 카운팅이 완벽하게 작동합니다.
* **🧹 구버전 대화방 파편화 로컬 데이터 자동 정화 마이그레이션**
  - 1:1 통합 채팅방 전환에 따라 브라우저 로컬 캐시(`localStorage`)에 남아있을 수 있는 구버전 다중 대화방 잔여 데이터를 앱 최초 로드 시 탐지하여 자동으로 필터링 및 클린 제거하는 마이그레이션 프로세스를 추가했습니다.
* **🏷️ 게스트 계정 닉네임 " (게스트)" 표기 일관성 보장**
  - 일회용 게스트로 로그인한 유저는 닉네임 뒤에 ` (게스트)` 접미사가 자동으로 결합되어 저장 및 렌더링됩니다.
* **🔐 게스트 로그인 계정 도용 차단 및 중복 닉네임 검증 구현**
  - 로그인 모달에서 게스트 로그인 시도 시, 가입 회원 계정(예: `간장` 등)이거나 현재 접속 중인 닉네임일 경우 로그인을 차단합니다.
* **🛠️ 내 글 수정 기능 추가**
  - 피드 카드 내 '수정' 버튼 및 '내 정보' 드로어 목록에 '수정' 버튼을 추가하고, 미니 스티커 도감 선택 패널이 장착된 수정 모달창을 구현했습니다.
* **💬 댓글 기능 추가 (아코디언 토글)**
  - 교환소 피드 카드 하단에 댓글 펼치기 버튼을 제공하고, 실시간 댓글 등록 및 삭제를 연동했습니다.
* **🚨 교환글 및 댓글 신고 기능 추가**
  - 부적절한 사유로 신고할 수 있는 신고 접수 팝업창을 구현했습니다.
* **👑 관리자 전용 대시보드 구현 (간장 계정 전용)**
